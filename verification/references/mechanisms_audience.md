## №27 — Моделирование аудитории {#27}

### Суть
Симуляция восприятия документа ключевыми стейкхолдерами: что вызовет вопросы, что покажется слабым, что может быть неверно интерпретировано.

### Когда применять
- Документы для руководства
- Презентации для конкретной аудитории
- Стратегические обоснования

### Маркировка: LLM-only

### Процедура

**Шаг 1. Определи стейкхолдеров**
Кто будет читать/слушать? Обычно 2–4 роли.

**Шаг 2. Для каждой роли — симуляция**
Задай: «Если бы я был [роль], что бы я спросил / заметил / потребовал?»

**Шаг 3. Типичные реакции по ролям**

| Роль | Типичные вопросы |
|------|-----------------|
| Руководитель | «Сколько стоит? Какие риски? Когда результат? Зачем мне это?» |
| Финансист | «Где разбивка по статьям? Почему не учтены начисления? Какой ROI?» |
| Юрист | «На каком основании? Есть ли НПА? Какие правовые риски?» |
| Оппонент | «А если не сработает? А альтернативы? А кто за это отвечает?» |
| PR/пресс-служба | «Можно ли эту фразу вырвать из контекста? Как это прозвучит в СМИ?» |

**Шаг 4. Фиксация**
```
Стейкхолдер: [роль]
Вероятные вопросы: [список]
Слабые места: [что потребует усиления]
Риск неверной интерпретации: [где и как]
```

### Шаблон фиксации
```
| Стейкхолдер | Вопрос | Слабое место | Рекомендация |
|-------------|--------|--------------|--------------|
```

### Пример — обоснование ЦПИКП для ректората МИФИ

> **Ректор:**
> - «Почему именно 23 человека, а не 15?» → нужна таблица функциональной нагрузки
> - «Какой ROI в первый год?» → нужен прогноз привлечения
> - «Что если KPI не выполнят?» → нужен сценарий с KPI=50%
>
> **Проректор по финансам:**
> - «192М — это с начислениями или без?» → уточнить: ФОТ = 128М, с начислениями (30.2%) = 167М
> - «Откуда деньги на переменную часть при 0% KPI?» → показать механизм финансирования
>
> **Юрист:**
> - «KPI-вестинг 69% переменной — соответствует ли ТК РФ?» → проверить ограничения

## №31 — Проверка адресата {#31}

### Суть
Язык, глубина, тон и формат документа соответствуют целевому получателю.

### Когда применять
- Любые документы с известным адресатом
- Презентации и выступления
- Официальные письма

### Маркировка: LLM-only

### Процедура

**Шаг 1. Зафиксируй адресата**
- Должность/роль
- Уровень погружения (эксперт / управленец / новичок)
- Контекст (совещание / выступление / чтение / подготовка)

**Шаг 2. Проверка по 4 осям**

| Ось | Вопрос |
|-----|--------|
| Язык | Понятен ли текст без словаря? Нет ли избыточного жаргона или упрощения? |
| Глубина | Достаточно ли деталей? Не утопает ли в мелочах? |
| Тон | Уместен ли стиль для данного адресата и ситуации? |
| Формат | Удобен ли для сценария использования? (чтение на ходу vs детальный анализ) |

**Шаг 3. Калибровка под известных адресатов**

| Адресат | Формат | Язык | Объём |
|---------|--------|------|-------|
| Руководство Росатом/МИФИ | Вывод первым абзацем, сравнительные таблицы | Деловой, без жаргона | 1–3 стр. |
| Сенатор Глебова Л.Н. | Тезисы, простой язык, крупный шрифт | Без англицизмов, пояснять термины | 2–4 стр. |
| Т-Банк DIGITAL | Слайды 16:9, жёлтая полоса с выводом | Финансовый, допустим жаргон | 12–16 слайдов |
| Рабочая группа | Таблицы, чеклисты, сроки, ответственные | Оперативный | Без ограничений |
| Аудитория Конгресса | «Проблема→решение→результат», 2–3 цифры | Публичный, эмоциональные якоря | Тезисы 1–2 стр. |

**Шаг 4. Фиксация**
```
Адресат: [кто]
Контекст: [как будет использоваться]
Язык: ✅/⚠️ [комментарий]
Глубина: ✅/⚠️ [комментарий]
Тон: ✅/⚠️ [комментарий]
Формат: ✅/⚠️ [комментарий]
```

### Шаблон фиксации
```
| Ось | Статус | Комментарий |
|-----|--------|-------------|
| Язык | ✅/⚠️ | |
| Глубина | ✅/⚠️ | |
| Тон | ✅/⚠️ | |
| Формат | ✅/⚠️ | |
```
