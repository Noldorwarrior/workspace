# Установка скилла верификации в Claude.ai проект

## Шаг 1. Создай проект
1. Открой claude.ai → Projects → «Create project»
2. Назови проект (например, «Рабочее пространство»)

## Шаг 2. Загрузи Project Instructions
1. Открой настройки проекта → «Set custom instructions»
2. Скопируй содержимое `SKILL.md` целиком
3. Вставь в поле Custom Instructions → Сохрани

## Шаг 3. Загрузи Project Knowledge
Загрузи все файлы из папки `references/` в Project Knowledge:
- mechanisms_factual.md
- mechanisms_logical.md
- mechanisms_source.md
- mechanisms_numerical.md
- mechanisms_document.md
- mechanisms_audience.md
- presets_matrix.md
- verification_self_mode.md

## Шаг 4. Скрипты (опционально)
Скрипты из `scripts/` используются только если в чате доступен computer use.
Если computer use доступен:
1. Загрузи все .py файлы из `scripts/` в Project Knowledge
2. В начале сессии Claude сам скопирует нужные скрипты и запустит

Если computer use НЕ доступен — скилл работает без скриптов.
Все скриптовые механизмы имеют LLM-fallback (ручная проверка по процедуре).

## Шаг 5. Проверь работу
Начни новый чат в проекте и напиши:
> «Какие механизмы верификации ты знаешь? Покажи пресет П1 Аналитик.»

Claude должен найти информацию через project_knowledge_search и ответить списком механизмов пресета П1.

## Готово!
Теперь при любой комплексной задаче Claude автоматически предложит план верификации.

## Примеры использования

### Пример 1: аналитическая записка
Пользователь: «Напиши аналитическую записку по молодёжной политике на основе трёх приложенных файлов»
→ Claude автоматически определяет: тип = аналитика, файлов = 3, пресет = П1 «Аналитик» + М2
→ Claude показывает план верификации через ask_user_input
→ После одобрения: создаёт записку → проводит верификацию → выдаёт результат + отчёт

### Пример 2: проверка чужого документа
Пользователь: «Проверь этот документ на ошибки и противоречия»
→ Claude определяет: тип = аудит, пресет = П13 «Аудитор»
→ Показывает план → получает одобрение → проводит проверку по механизмам
→ Выдаёт отчёт с таблицей найденных проблем

### Пример 3: презентация
Пользователь: «Сделай презентацию по отчёту»
→ Claude определяет: тип = презентация, пресет = М4 «Презентационная» + элементы П7
→ Создаёт pptx → проверяет формат, согласованность, адресат
→ После согласования — создаёт HTML-версию → проверяет sync pptx↔html

### Пример 4: ручной вызов верификации
Пользователь: «Проведи верификацию П3 Бухгалтер для этого xlsx»
→ Claude сразу открывает presets_matrix → находит механизмы П3
→ Показывает план верификации → после одобрения выполняет

### Пример 5: отказ от верификации
Пользователь: «Без верификации»
→ Claude не проводит верификацию, приступает к задаче напрямую
