# ПРОМТ: Интерактивный справочный сайт системы верификации

> **Исполнитель:** Claude Code  
> **Формат выхода:** один standalone HTML-файл `verification_dashboard.html`  
> **Язык интерфейса:** русский  
> **Язык кода:** английский (комментарии — русский)

---

## ЗАДАЧА

Создай максимально насыщенный интерактивный справочный сайт (SPA) по системе верификации. Один HTML-файл, работает офлайн, без внешних зависимостей кроме CDN-библиотек.

**Уровень качества:** суперпремиум. Это НЕ дашборд в привычном смысле — это полноценный информационный портал с инфографикой, анимациями, интерактивными инструментами и визуализацией данных уровня, который можно было бы показать на конференции или использовать как обучающий материал для команды.

## ДИЗАЙН-НАПРАВЛЕНИЕ

**Luxury dark dashboard** с элементами editorial design:

### Визуальный язык
- Основа: тёмный фон (#06080f → #0f172a) с animated mesh gradient (3 orba с blur)
- Карточки: glassmorphism (backdrop-filter: blur, полупрозрачные фоны, тонкие borders)
- Акценты: gradient от indigo (#6366f1) через purple (#8b5cf6) к cyan (#06b6d4)
- Типографика: display-шрифт для заголовков (Syne 800), body-шрифт (DM Sans)
- Заголовки страниц: gradient text (background-clip: text)

### Анимации
- **Page transitions:** fade-in + translateY при переключении страниц
- **Scroll-triggered:** IntersectionObserver — элементы появляются при скролле (staggered delays)
- **Counters:** анимированный подсчёт чисел (0 → 32) при появлении в viewport
- **Hover micro-interactions:** translateY(-3px), box-shadow glow, border-color shift на каждой карточке
- **Expandable cards:** slideDown с анимацией при раскрытии
- **Chart animations:** Chart.js built-in анимации при первом рендере
- **Skeleton loading:** pulse-анимация при переключении страниц (опционально)

### UX-принципы
- Первый экран — hero с ключевыми числами и визуалом
- Sidebar навигация (сворачиваемая на мобильном)
- Глобальный поиск по ВСЕМУ контенту (механизмы, пресеты, глоссарий, FAQ, скрипты)
- Breadcrumbs или back-navigation
- Состояние фильтров сохраняется при переключении разделов

- Шаг 2: Результат — анимированное появление рекомендации с:

  - Список механизмов с бейджами
  - Скрипты

  - Кнопка «Показать карточку пресета» → скролл вниз

**Табы: Малая / Полная / Все**

**Карточки пресетов (раскрываемые):**
Закрытая: ID + название + целевая задача + mini progress bar (соотношение 🔧/🧠)
Открытая:
- Описание
- Stacked bar: скриптовые vs LLM (с числами и процентами)
- Полный список механизмов (бейджи, кликабельные → переход к карточке механизма)

- Режим агента (иконка + описание)

- Animated tree diagram: М1 → М2 → М3, М4, М5 (D3 или CSS)

### 5. 📊 Матрица покрытия (`#matrix`)

**Интерактивная таблица 32×19:**
- Sticky headers (и вертикальные, и горизонтальные)
- Hover на ячейку → подсветка всей строки + всего столбца (crosshair)
- Клик на ячейку → тултип с деталями (механизм + пресет)
- Цветовая градация (количество общих механизмов между пресетами)
- Строка итогов (количество механизмов в пресете)
- Столбец итогов (количество пресетов у механизма)

**Фильтры матрицы:**
- Подсветить конкретный пресет (вся колонка горит)
- Подсветить конкретный механизм (вся строка горит)
- «Сравнить 2 пресета» — показать только их колонки + diff (общие / уникальные)

**Chord diagram (D3):**
- Связи между пресетами через общие механизмы
- Толщина связи = количество общих механизмов
- Hover → подсветка конкретных связей

## JSON-ДАННЫЕ

Все данные — один объект `const DATA = {...}` внутри `<script>`:

```javascript
const DATA = {
  mechanisms: [...],    // 32 объекта: id, name, type, category, script, presets[], desc, oneliner, howItWorks, exampleError, exampleOk, limitations
  presets: [...],       // 19 объектов: id, name, level, mechs[], scripts[], mode, target, desc, relatedPresets
  scripts: [...],       // 11 объектов: name, desc, files, mechs, type, args, usage, dependencies
  glossary: [...],      // 20+ объектов: term, def, analogy, example
  workflow: [...],      // 5 объектов: num, title, icon, who, desc, detail
  agents: [...],        // 4 объекта: mode, name, icon, when, reliability, cost, bias, checkers, presets
  scenarios: [...],     // 5 объектов: title, icon, preset, scripts, llm, finds, fullWorkflow
  faq: [...],           // 15 объектов: q, a, group
  refs: [...],          // 6+4 объектов: file, mechs, theme
  matrix: [...]         // 32×19 boolean matrix (сгенерируй из presets.mechs)
};
```

**Данные берутся из файлов проекта.** Прочитай ВСЕ reference-файлы для заполнения полей `howItWorks`, `exampleError`, `exampleOk`, `limitations` у каждого механизма.

## ПРАВИЛА ВЫПОЛНЕНИЯ

1. **Один файл.** Всё в одном HTML. Inline CSS + JS. Внешние — только CDN-библиотеки.
2. **Сначала прочитай все файлы проекта.** Не начинай писать код без полной картины данных.
3. **Данные из проекта — не выдумывай.** Каждый текст, число, пример — из reference-файлов или СПРАВКА_ВЕРИФИКАЦИЯ.md.
4. **Антикраш.** Если файл становится слишком большим (>5000 строк) — разумно сократи тексты, но НЕ убирай функционал.
5. **Тестируй.** После создания файла открой его и проверь, что все страницы рендерятся, графики работают, фильтры фильтруют, поиск ищет.
6. **Русский интерфейс.** Все лейблы, тултипы, плейсхолдеры — на русском.
7. **Файл сохрани** в корень проекта как `verification_dashboard.html`.
